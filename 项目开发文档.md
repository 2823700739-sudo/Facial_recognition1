# 人脸隐私加密便签 — 项目架构与技术栈深度解析文档

本文档旨在帮助你更好地理解当前项目经过重构与升级后的整体架构、工作原理以及所使用的技术栈。

## 一、 项目整体架构 (Client-Server 模式)

本项目已经从本地单机应用成功升级为 **Client-Server (C/S，客户端-服务端)** 架构。
这种架构将“界面展示与设备交互”跟“数据存储与核心逻辑”物理隔离，大大提升了数据的安全性与系统的扩展性。

*   **客户端 (Client - `project` 文件夹)**: 
    *   负责呈现 UI 界面（登录、注册、主界面便签管理）。
    *   负责调用本地硬件（通过摄像头采集人脸图像）。
    *   负责调用人脸识别算法提取特征值。
    *   将所有用户的意图（注册新用户、验证登录、新建/修改/搜索/删除便签）封装成 **JSON** 数据包，通过 **TCP/IP Socket** 发送给服务端，并同步等待服务端的响应。
*   **服务端 (Server - `server` 文件夹)**:
    *   运行在后台（或另一台机器上，本项目目前设为 `192.168.11.84:7777` 监听）。
    *   持有一个中心化的 **SQLite 数据库 (`note_db.db`)**。
    *   监听并接收多个客户端的连接。
    *   解析客户端发来的 JSON 请求，执行相应的 SQL 语句，并将结果（成功或失败、便签数据列表等）打包成 JSON 返回给客户端。
    *   拥有一个 `admin_win` 监控界面，可以实时查看当前连接的客户端 IP、端口以及服务器运行日志。

## 二、 核心技术栈详细说明

本项目是基于 C++ 及 Qt 框架开发的中级综合性项目，具体技术栈如下：

### 1. C++ 11/14 (核心编程语言)
*   **用途**: 支撑整个客户端与服务端的基础逻辑。
*   **特性使用**: 使用了面向对象编程 (OOP) 思想，封装了各个窗口类；使用了指针、引用、STL 容器（如局部缓存）；结合 Qt 使用了 Lambda 表达式处理信号槽等。

### 2. Qt 5.x 框架 (UI 与 应用核心模块)
Qt 是本项目最核心的框架，几乎贯穿了所有的功能模块。
*   **Qt Widgets (`QWidget`, `QMainWindow`, `QDialog`)**: 用于构建所有的图形用户界面（登录界面、注册界面、人脸识别遮罩界面、主便签界面、服务端监控界面）。使用了 QSS (Qt Style Sheets) 进行界面美化，如锁屏时的半透明黑色蒙版 (`background-color: rgba(0, 0, 0, 200);`)。
*   **Qt Network (`QTcpServer`, `QTcpSocket`)**: 
    *   **服务端 (`QTcpServer`)**: 在特定端口 (7777) 监听，一旦有新的客户端 (`nextPendingConnection`)，就建立连接并维护 socket 通信。
    *   **客户端 (`QTcpSocket`)**: 启动时向服务器发起连接请求。为了简化逻辑，客户端封装了一个 `sendSyncRequest` 发送同步请求，通过 `waitForReadyRead` 阻塞等待服务器回包（ JSON 数据）。
*   **Qt Sql (`QSqlDatabase`, `QSqlQuery`)**: 
    *   目前完全移交给了**服务端**。负责与底层的 SQLite 数据库文件通信，执行 `CREATE`, `SELECT`, `INSERT`, `UPDATE`, `DELETE` 等 SQL 命令，使用了 `LIKE` 实现了便签的**模糊搜索**功能。
*   **Qt Core (JSON 模块: `QJsonDocument`, `QJsonObject`, `QJsonArray`)**: 
    *   作为客户端和服务端之间通信的“共同语言”。将复杂的指令（如操作类型 action、用户名、特征值数组、便签内容）序列化为文本字节流传输，接收方再反序列化还原为 JSON 对象。

### 3. OpenCV 4.5.x (计算机视觉库)
*   **用途**: 用于客户端打开电脑物理摄像头（`cv::VideoCapture`），实时读取视频流帧（`cv::Mat`）。
*   **图像转换**: 由于 OpenCV 读取的画面为 BGR 格式，而 Qt 显示需要 RGB，项目中使用 `cvtColor` 等方法将 `cv::Mat` 转为 `QImage` 并展示在 UI的 `QLabel` 上。
*   *(注: 历史上曾使用 Haar 级联分类器做人脸框检测，现可能结合 SeetaFace 共同工作)*。

### 4. SeetaFace (中科视拓 - 开源人脸识别引擎)
*   **用途**: 本项目的高级生物识别核心。替代了传统的账号密码。
*   **人脸检测 (Face Detection)**: 从摄像头截取的画面中，精准定位人脸的位置和大小。
*   **特征提取 (Feature Extraction)**: 将人脸图像转换为一个高维度的数学向量（特征值 / Feature Array）。在这套系统中，注册时将特征值存入数据库；
*   **特征比对 (Face Verification/Identification)**: 登录或“解锁 (Unlock Mode)” 时，提取当前人脸特征，与服务器拉取回来的特征库进行余弦相似度计算，相似度大于设定阈值（如 > 0.6）即判定为本人。

### 5. SQLite 3 (轻量级本地数据库)
*   **用途**: 关系型数据存储，无需额外部署 MySQL 这样的重型服务。
*   **数据表结构**:
    *   `users`: 记录基本用户信息 `user_id`, `username`。
    *   `user_faces`: 专门隔离存储用户的人脸特征数组文本 `face_feature`，并关联 `user_id`。
    *   `notes`: 存储用户的便签信息，包括 `title`, `content`, `create_time`, `update_time` 等，强制关联特定的 `user_id` 以保障数据隔离。
    *   `sessions` (可选): 用于记录客户端的登录状态与时长。

## 三、 模块工作流分析 (以“解锁便签”为例)

1. **触发**: 用户在主界面 (`main_win`) 点击“解锁”按钮。
2. **UI 限制**: 客户端弹出一个含有透明蒙版的模态对话框 `face_in`（传入当前 `target_user_id` 和 `unlock_mode` 标志），遮盖背景防止偷窥。
3. **硬件调用**: OpenCV 启动摄像头，将画面实时渲染到界面。
4. **算法分析**: SeetaFace 捕获当前帧，如果检测到人脸，提取特征值。
5. **网络通信 & 逻辑判断**: 
    * 客户端早已通过 `load_features` 的 JSON 请求从服务器获取了当前 `target_user_id` 的合法特征值。
    * 客户端对比当前人脸与合法特征值，如果匹配，触发 `loginSuccessful` 信号。
6. **解锁完成**: `main_win` 接收到信号，关闭 `face_in` 窗口，移除蒙版，恢复便签查看和编辑功能。

通过这套架构，你成功分离了前后端，掌握了 C++ GUI、网络并发、数据库操作以及计算机视觉 SDK 整合这几大核心技能。
1. **通信协议**：基于TCP协议的自定义二进制协议
2. **数据格式**：包头+包体结构，支持数据分包
3. **心跳机制**：定时发送心跳包，检测连接状态
4. **重连机制**：客户端断线自动重连服务端

### 3.5 加密&辅助技术
1. 自定义异或加密：实现文字、图片Base64数据加密解密；
2. Qt Base64编码：截图转文本存入数据库；
3. QTimer定时器：实现实时人脸检测、自动倒计时锁定；
4. 单一人脸认证机制：全程依靠高精度人脸特征提取进行身份匹配，无感且安全。

## 六、客户端-服务端功能模块拆解
### 模块1：客户端网络通信模块
1. TCP连接管理：基于Qt Network连接服务端、断线重连、心跳检测；
2. 数据分包处理：处理TCP数据分包和粘包问题；
3. 请求响应处理：封装网络请求，处理服务端响应；
4. 数据同步机制：本地缓存与服务端数据同步。

### 模块2：客户端单一人脸认证模块
1. 首次启动连接服务端，引导用户注册基本信息（用户名+人脸特征）；
2. 人脸注册：采集人脸特征，加密后直接绑定账号发送到服务端；
3. 人脸识别解锁：登入时实时检测人脸，服务端比对特征验证通过即登录；
4. 会话管理：按提取特征身份匹配后维护与服务端的登录会话。

### 模块3：客户端便签管理模块
1. 便签编辑：支持文字编辑、格式设置；
2. 截图功能：屏幕截图，Base64编码加密；
3. 数据同步：便签实时同步到服务端；
4. 离线缓存：网络断开时本地缓存数据。

### 模块4：客户端安全锁定模块
1. 人脸检测：实时摄像头监控，3秒无人脸自动锁定；
2. 数据保护：锁定后清空本地敏感数据；
3. 会话失效：锁定后通知服务端会话失效。

### 模块5：服务端网络服务模块
1. TCP服务器：基于Qt Network多线程处理客户端连接；
2. 协议解析：解析TCP数据包，验证数据格式；
3. 会话管理：维护用户登录状态，处理会话超时；
4. 连接管理：支持多客户端并发访问。

### 模块6：服务端业务处理模块
1. 用户管理：用户注册、登录验证、权限控制；
2. 人脸验证：接收人脸特征，进行比对验证；
3. 数据存储：加密存储用户数据到数据库；
4. 数据同步：处理客户端数据同步请求。

### 模块7：服务端安全管理模块
1. 数据加密：服务端数据加密存储；
2. 访问控制：基于用户权限的数据访问控制；
3. 日志记录：记录用户操作和系统事件；
4. 异常处理：处理异常请求，防止恶意攻击。
