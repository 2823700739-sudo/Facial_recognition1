# 人脸隐私加密便签—项目开发详细文档
## 一、项目基础信息
1. 开发语言：C++11（客户端+服务端）
2. UI框架：Qt 5.15.2（客户端）
3. 视觉库：OpenCV 4.5.5（Haar人脸检测，客户端）
4. 数据库：SQLite3（服务端）
5. 网络通信：TCP/IP Socket通信
6. 项目难度：中级（增加网络通信）
7. 开发周期：5~7天
8. 运行模式：客户端-服务端架构，数据集中管理

## 二、项目创新亮点
1. 客户端-服务端架构，数据集中管理更安全；
2. 基于人脸特征的单一高安全认证机制，无需记忆密码；
3. 支持文字笔记+屏幕截图双内容加密存储；
4. 3秒无人脸自动锁定，防偷窥隐私泄露；
5. 仅存储人脸特征值，不保存人脸原图，合规安全；
6. 客户端数据实时同步到服务端，支持多设备使用；
7. 服务端数据加密存储，客户端无敏感数据残留；
8. 彻底摒弃传统账号密码，杜绝密码泄露和遗忘风险。

## 五、服务端数据库设计
### 5.1 数据库文件名
server_note_db.db（服务端）

### 5.2 数据表1：users（存储用户信息）
字段设计：
user_id INTEGER PRIMARY KEY AUTOINCREMENT
username TEXT UNIQUE 唯一用户名
create_time TEXT 账号创建时间
last_login TEXT 最后登录时间
status INTEGER 用户状态（0-正常，1-禁用）

建表SQL：
CREATE TABLE IF NOT EXISTS users (
user_id INTEGER PRIMARY KEY AUTOINCREMENT,
username TEXT UNIQUE NOT NULL,
create_time TEXT NOT NULL,
last_login TEXT,
status INTEGER DEFAULT 0
);

### 5.3 数据表2：user_faces（存储用户人脸特征）
字段设计：
face_id INTEGER PRIMARY KEY AUTOINCREMENT
user_id INTEGER 关联用户ID
face_feature TEXT 加密后人脸特征字符串
create_time TEXT 人脸注册时间

建表SQL：
CREATE TABLE IF NOT EXISTS user_faces (
face_id INTEGER PRIMARY KEY AUTOINCREMENT,
user_id INTEGER NOT NULL,
face_feature TEXT NOT NULL,
create_time TEXT NOT NULL,
FOREIGN KEY(user_id) REFERENCES users(user_id)
);

### 5.4 数据表3：notes（存储用户便签）
字段设计：
note_id INTEGER PRIMARY KEY AUTOINCREMENT
user_id INTEGER 关联用户ID
title TEXT 便签标题
content TEXT 加密文字内容
image_data TEXT 截图Base64加密数据
create_time TEXT 创建时间
update_time TEXT 最后修改时间

建表SQL：
CREATE TABLE IF NOT EXISTS notes (
note_id INTEGER PRIMARY KEY AUTOINCREMENT,
user_id INTEGER NOT NULL,
title TEXT,
content TEXT,
image_data TEXT,
create_time TEXT NOT NULL,
update_time TEXT NOT NULL,
FOREIGN KEY(user_id) REFERENCES users(user_id)
);

### 5.5 数据表4：sessions（存储用户会话）
字段设计：
session_id INTEGER PRIMARY KEY AUTOINCREMENT
user_id INTEGER 用户ID
session_token TEXT 会话令牌
login_time TEXT 登录时间
expire_time TEXT 过期时间
client_ip TEXT 客户端IP

建表SQL：
CREATE TABLE IF NOT EXISTS sessions (
session_id INTEGER PRIMARY KEY AUTOINCREMENT,
user_id INTEGER NOT NULL,
session_token TEXT UNIQUE NOT NULL,
login_time TEXT NOT NULL,
expire_time TEXT NOT NULL,
client_ip TEXT,
FOREIGN KEY(user_id) REFERENCES users(user_id)
);

## 六、客户端-服务端功能模块拆解
### 模块1：客户端网络通信模块
1. TCP连接管理：基于Qt Network连接服务端、断线重连、心跳检测；
2. 数据分包处理：处理TCP数据分包和粘包问题；
3. 请求响应处理：封装网络请求，处理服务端响应；
4. 数据同步机制：本地缓存与服务端数据同步。

### 模块2：客户端单一人脸认证模块
1. 首次启动连接服务端，引导用户注册基本信息（用户名+人脸特征）；
2. 人脸注册：采集人脸特征，加密后直接绑定账号发送到服务端；
3. 人脸识别解锁：登入时实时检测人脸，服务端比对特征验证通过即登录；
4. 会话管理：按提取特征身份匹配后维护与服务端的登录会话。

### 模块3：客户端便签管理模块
1. 便签编辑：支持文字编辑、格式设置；
2. 截图功能：屏幕截图，Base64编码加密；
3. 数据同步：便签实时同步到服务端；
4. 离线缓存：网络断开时本地缓存数据。

### 模块4：客户端安全锁定模块
1. 人脸检测：实时摄像头监控，3秒无人脸自动锁定；
2. 数据保护：锁定后清空本地敏感数据；
3. 会话失效：锁定后通知服务端会话失效。

### 模块5：服务端网络服务模块
1. TCP服务器：基于Qt Network多线程处理客户端连接；
2. 协议解析：解析TCP数据包，验证数据格式；
3. 会话管理：维护用户登录状态，处理会话超时；
4. 连接管理：支持多客户端并发访问。

### 模块6：服务端业务处理模块
1. 用户管理：用户注册、登录验证、权限控制；
2. 人脸验证：接收人脸特征，进行比对验证；
3. 数据存储：加密存储用户数据到数据库；
4. 数据同步：处理客户端数据同步请求。

### 模块7：服务端安全管理模块
1. 数据加密：服务端数据加密存储；
2. 访问控制：基于用户权限的数据访问控制；
3. 日志记录：记录用户操作和系统事件；
4. 异常处理：处理异常请求，防止恶意攻击。
